# H.Studio — Harvey Specter

> A full-service creative studio: branding, web design and engineering.

## Brand Identity
- **Personality:** Bold, editorial, minimal. Big typography, black & white, lots of breathing room.
- **Colors** (defined as variables in `app/globals.css`):
  - Background: warm off-white `#f2f2f0` (`--c-paper`)
  - Text / buttons: black `#000000` (`--accent`)
  - Body copy: near-black `#1f1f1f` (`--muted`)
  - On imagery: white `#ffffff` (`--on-dark`)
- **Fonts:**
  - `Inter` — all headings and text
  - `Geist Mono` — the bracket labels like `[ Hello i'm ]`
  - `Playfair Display` (italic) — the decorative serif "&" in big statements

## Pages
- **Homepage** (`/`) — built in the **Page Builder** (the "Home" page), in order: **Menu → Hero → Creative Statement → About → Photo Banner → Services → Selected Work → Testimonials → News → Footer**. (Complete.)
- **Projects** (`/projects`) — built in the Page Builder: **Menu → Projects Hero (cover) → Selected Work → Statement Hero (dark closing CTA) → Footer**.
- **Contact** (`/contact`) — built in the Page Builder: **Menu → Contact (form) → Footer**. The form posts to `/api/contact` and saves each message to Sanity — read them under **Messages** in the Studio.
- **Services** (`/services`) — built in the Page Builder: the (reused) **Projects Hero** + one **Service Feature** section per service, alternating light/dark and image side. A starting point — refine it in the builder.
- **New pages** — anything you build in the Page Builder with another web address opens at `/that-address`.
- **Component Library** (`/lab`) — *internal only.* A living catalogue of every section and UI primitive, each rendered live in an isolated frame with its name + props (desktop/mobile toggle). **Not indexed** by search engines: `noindex` on the page + `Disallow: /lab` in `robots.txt`. Metadata lives in `lib/showcase.ts`; isolated previews at `/lab/preview/[slug]`.

## Page Builder (Sanity CMS) — READ THIS FIRST
**Your whole site is now built like Webflow: drag-and-drop sections that you edit right on the page.** Everything happens inside Sanity — no code.

- **Where to edit:** open **`/studio`** in your site (e.g. `yoursite.com/studio`). Log in with your Sanity account. The sidebar (**Website Content**) has:
  - **Site Settings (menu & footer)** — the **global** menu and footer. Set the logo, menu links and footer here **once**; every page uses them automatically, so you never re-link the menu when you add a page.
  - **Pages** — the builder. Open **Home** (or click **Create** for a new page). Each page has a **Page name**, a **Web address** (use `home` for the main page; anything else becomes `/that-address`), a **Menu color** (white or black, for that page's top), and **Sections**.
  - **Services · Work (Projects) · News Posts** — the three content lists your sections pull from.
  - **Messages** — the inbox of contact-form submissions (read-only; new ones arrive automatically when someone uses the Contact page).
- **Building a page:** in **Sections**, click **Add item** to drop in a section, **drag** the sections to reorder them, and click one to edit its text. (The menu and footer are **not** sections anymore — they come from Site Settings automatically.) The available sections are the real components of the site:
  - **Hero (opening)** — big name, small label, intro paragraph, button, background photo.
  - **Projects Hero (cover)** — editorial portfolio cover: two-word title (second word outlined), intro + category chips, and a scrolling marquee. Light/paper.
  - **Statement Hero (dark)** — dark, high-contrast hero with a giant outlined "ghost" word behind a centered statement (type `&` for the italic symbol), intro and an outlined button. Great as a page opener or a closing CTA.
  - **Creative Statement** — the big editorial lines. Type `&` and it becomes the fancy italic symbol. ~5 lines.
  - **About** — intro paragraph + tall portrait.
  - **Photo Banner** — a full-width photo.
  - **Services / Work / News sections** — placement blocks: they automatically show your Services / Work / News lists (edit those in the sidebar). News shows the first 5.
  - **Service (feature section)** — a big, self-contained section for ONE service (image + content, "includes" list, CTA). Light or dark theme, image left or right. Add one per service and alternate them.
  - **Testimonials** — the reviews are edited **inside this block** (quote, author, logo); order sets the desktop layout.
  - **Contact (form)** — big statement + direct details on the left, a working form on the right. Editable: eyebrow, heading, intro, email, location, availability, socials, form chip options, and the success message. Submissions go to **Messages**.
  - **Footer** — the "Have a project in mind?" text, button + link, social/legal links, giant wordmark.
- **Edit visually (recommended):** in **Pages → Home**, use the **Presentation** view to see the live site next to the editor and **click text on the page to edit it**.
- **After editing:** click **Publish**. The website updates within seconds.
- **The 3 lists (Services / Work / News):** open the list in the sidebar, click **Create**, fill it in, set the **Order** number (lower = first), and **Publish**.
- **What stays in code (by design):** the three section headings — **Services / Deliverables**, **Selected Work**, and the **News** heading — plus the "Testimonials" wordmark and the site colors.
- **Live preview (click-to-edit) needs a token:** add a Sanity **read token** as `SANITY_API_READ_TOKEN` in your environment. Without it the published site works fine — only the click-to-edit preview is off. (Create one at sanity.io → your project → API → Tokens, with **Viewer** access.)
- **Under the hood:** the `page` document + section blocks live in `sanity/schemaTypes/` (`documents/page.ts`, `blocks/*.ts`); the sidebar is `sanity/structure.ts`; pages are fetched in `sanity/lib/queries.ts` (`PAGE_QUERY`) and rendered by `components/PageBuilder.tsx` from `app/page.tsx` (home) and `app/[slug]/page.tsx` (other pages). Visual editing is wired via `presentationTool` in `sanity.config.ts`, `sanity/lib/live.ts` (`sanityFetch`/`SanityLive`), `app/api/draft-mode/*`, and `<VisualEditing/>` in `app/layout.tsx`. Images come from the Sanity CDN (`cdn.sanity.io`, allowed in `next.config.ts`). If the CMS is empty, the homepage falls back to the built-in defaults. The content import is `scripts/seed.ts` (re-run with `npx sanity exec scripts/seed.ts --with-user-token`).

## Components
- **ProjectsHero** (`components/ProjectsHero.tsx`) — the editorial "cover" for the projects page. Paper background, a giant two-word solid-black title, an index paragraph + category chips (same **glass tag** style as the Work cards), and a full-bleed **marquee ticker** of the disciplines (CSS `hero-marquee` keyframes in `globals.css`). The **project count is dynamic** — `count(*[_type=="project"])` in the query, so it grows automatically as you add projects. Masked line-reveal on load (GSAP), motion-safe.
- **StatementHero** (`components/StatementHero.tsx`) — a dark, high-contrast hero: a giant outlined **ghost word** drifts behind (GSAP scroll parallax) a centered statement (with the Playfair italic `&`), an intro and an outlined CTA. Reveals on load.
- **Grain** (`components/ui/Grain.tsx`) — reusable film-grain overlay (inline SVG noise), `blend="multiply"` on light surfaces, `screen` on dark.
- **Contact** (`components/Contact.tsx`) — the contact section: an editorial split with a big statement + direct details (email, location, socials) on the left and a working form on the right (underline fields, project-type chips, inline validation, idle → submitting → success states). Posts to `app/api/contact/route.ts`, which saves a `contactMessage` in Sanity (needs `SANITY_API_WRITE_TOKEN`; the form still succeeds gracefully without it).
- **PageBuilder** (`components/PageBuilder.tsx`) — the bridge between the CMS and the site. It takes a page's ordered list of section blocks and renders each one with the matching ready-made component (Navbar, Hero, About, Services, …). This is what makes the drag-and-drop builder work.
- **Footer** (`components/Footer.tsx`) — black footer with a "Have a project in mind?" CTA + outlined "Let's talk" button, social links (two aligned columns on desktop, stacked on mobile), a divider, legal links, and a giant "H.Studio" wordmark that bleeds off the bottom edge with a vertical "Coded By Claude" credit. Has `id="contact"`, so the navbar's Contact link scrolls here. Social/legal hrefs are placeholders (`#`) to fill in later.
- **SmoothScroll** (`components/SmoothScroll.tsx`) — wraps the whole page in GSAP **ScrollSmoother** so scrolling glides with a smooth, eased feel (desktop wheel/trackpad only; touch devices keep native scrolling; off for "reduce motion"). In-page anchor links (About, Services…) are intercepted so they scroll smoothly to their section.
- **Navbar** (`components/Navbar.tsx`) — top navigation, now a standalone **Menu block** the PageBuilder overlays on the top of the page (no longer rendered inside the Hero). Has a `theme` prop (`onDark` = white text, `onLight` = black text) so it reads on both photo/dark and light/paper backgrounds. On desktop: logo, links (About, Services, Projects, News, Contact) and a "Let's talk" button. On mobile: logo + a hamburger that opens a menu. Built with proper HTML tags (`<header>`, `<nav>`, `<ul>`, `<a>`, `<button>`).
- **Hero** (`components/Hero.tsx`) — full-screen opening with the big "Harvey Specter" name over a background photo, an intro paragraph and a "Let's talk" button. Has a progressive blur fading in at the bottom. (The menu is a separate Menu block overlaid on top by the PageBuilder.)
- **Button** (`components/ui/Button.tsx`) — the reusable black pill button used for every "Let's talk".
- **ProgressiveBlur** (`components/ProgressiveBlur.tsx`) — reusable CSS-only blur that gets stronger toward the bottom edge.
- **CreativeStatement** (`components/CreativeStatement.tsx`) — the big editorial "A creative director / Photographer / Born & raised on the south side of chicago." statement. Staggered/indented on desktop, centered on mobile, with a fade-up reveal when scrolled into view.
- **News** (`components/News.tsx`) — the "Keep up with my latest news & achievements" blog section (grey `#f3f3f3`). Desktop: the section **pins** to the viewport and the whole row (the vertical rotated heading + cards) **scrolls horizontally as you scroll the page** (GSAP ScrollTrigger pin + scrub, set up with `gsap.matchMedia` so it's desktop-only); alternate cards are staggered down. Mobile: a plain **Swiper** carousel. CMS-ready via an `items` prop (`lib/content.ts`), meant to be capped at 5 posts.
- **Testimonials** (`components/Testimonials.tsx`) — a giant "Testimonials" wordmark with review cards scattered around it. On desktop the cards are **draggable** (GSAP Draggable + inertia) and one sits *behind* the word while the rest are in front (z-index matches the Figma); on mobile it becomes a **Swiper** slider. CMS-ready via an `items` prop (see `lib/content.ts`); desktop positions cycle through a `POSITIONS` preset table in the component. Uses `swiper` + GSAP `Draggable`/`InertiaPlugin`.
- **ServiceFeature** (`components/ServiceFeature.tsx`) — a big, self-contained section for one service: an editorial image + content split with a numbered "includes" list and a CTA. `theme` (light/dark) and `imageSide` (left/right) props so you can alternate sections for rhythm. Reveals on scroll (GSAP). Used to build the `/services` page (one per service).
- **Works** (`components/Works.tsx`) — the "Selected Work" (004) portfolio section: a staggered two-column masonry on desktop, single column on mobile, closed by a corner-framed "Let's talk" CTA. Each card has an image with glass tags, a title and a ↗ arrow; cards lift/zoom on hover and reveal on scroll (GSAP). CMS-ready — takes an `items` prop (see `lib/content.ts`).
- **Services** (`components/Services.tsx`) — the dark "Services / [4] Deliverables" section. CMS-ready: takes an `items` prop (defaults to the placeholder list in `lib/content.ts`); the "[4]" count is derived from the number of items. Each row: an index `[ n ]` + divider line, then an italic title with a description and a 151px square thumbnail (side-by-side on desktop, stacked on mobile). Rows fade up on scroll (GSAP). Descriptions are placeholders in `DESCRIPTION`; thumbnails live at `public/service-1..4.png`.
- **PhotoBanner** (`components/PhotoBanner.tsx`) — a full-width (edge-to-edge) photograph of Harvey shooting with a camera. The image is oversized and clipped by the frame, and drifts vertically on scroll for a **GSAP parallax** effect (turns off for "reduce motion"). Photo lives at `public/photographer.png`.
- **About** (`components/About.tsx`) — the "002 / About" section: an intro paragraph framed by small corner ticks, paired with a tall black & white portrait. On desktop the label sits top-left, the framed paragraph is bottom-aligned with the portrait, and "002" pins to the portrait's top; on mobile everything stacks. Reveals on scroll with GSAP: text fades up, corner ticks pop in, and the portrait wipes open top-to-bottom with a slow zoom. Portrait lives at `public/about-portrait.png`.

## Design System (for future sections)
All styling uses Tailwind + variables in `app/globals.css`, so new pages stay consistent:
- Colors: `bg-paper`, `text-foreground`, `text-muted`, `text-on-dark`, `bg-accent`
- Side padding (page gutter): `px-[var(--gutter)]` (16px mobile → 32px desktop)
- Display title size: `text-[length:var(--text-hero)]`
- Labels: `font-mono text-[length:var(--text-label)]`

## How to Customize
- **Change any text or photo:** go to **`/studio` → Pages → Home**, click a section, edit it, and **Publish**. See "Page Builder" above.
- **Build a new page:** in **`/studio` → Pages**, click **Create**, set a **Web address**, add sections, drag to reorder, and **Publish**. It opens at `/that-address`.
- **Add/remove a section:** open the page, in **Sections** click **Add item** (or the trash icon), then drag to reorder.
- **Change a color:** edit the variables at the top of `app/globals.css` (e.g. `--c-paper`, `--accent`). (Colors are in code, not the CMS.)
- **Add a new service / project / news post:** in `/studio`, open that list, click **Create**, fill it in, set the **Order** number, and Publish.
- **Add/edit a testimonial:** open **Pages → Home → Testimonials** and edit the reviews inside that block.
- **Reorder list items:** change the **Order** number (lower = first) and Publish.
